Subnet Mask

A 32-bit number that partitions an IP address into its network portion and host portion, enabling subnetting, routing, and efficient IP address allocation.

IPv4CIDRAddressingNetworking
📖

What is a Subnet Mask?

A subnet mask is a 32-bit number written alongside an IP address. It uses binary 1s to mark the network bits and binary 0s to mark the host bits. When logically ANDed with an IP address, the result is the network address of that subnet.

For example, IP 192.168.1.45 with mask 255.255.255.0 → network address 192.168.1.0, host .45.

Why Use Subnetting?

Reduce Congestion
Limits broadcast domains so broadcast traffic only floods a smaller subnet, not the entire network.
Security Isolation
Finance, HR, and Dev departments can be placed in separate subnets with firewall rules between them.
Efficient IP Use
Assign only the address block size needed, avoiding waste. A /30 for a P2P link instead of a /24.
Easier Management
Smaller logical segments are easier to monitor, troubleshoot and apply access policies to.

Channel Allocation Problem

Managing how a shared communication channel's bandwidth is divided among multiple competing stations — the core challenge of the MAC sublayer.

MACALOHACSMAMultiplexing
📖

The Core Problem

When multiple stations share one physical channel, simultaneous transmissions cause collisions, destroying both frames. A channel allocation protocol defines rules for who may transmit and when, balancing throughput, fairness, and latency.

ALOHA Throughput Curve

⚙️

CSMA Variants Explained

  • 1-persistent CSMA: If channel idle → transmit immediately. If busy → keep listening and transmit as soon as it becomes idle. Highest collision risk.
  • Non-persistent CSMA: If busy → wait a random time, then check again. Less collision probability but lower channel utilisation.
  • p-persistent CSMA: If idle → transmit with probability p; defer with probability (1-p). Balance between the above two.
  • CSMA/CD (Collision Detection): Used in wired Ethernet. While transmitting, monitor for collision (voltage spike). If detected → stop, send 48-bit jam signal, wait exponential backoff, retry.
  • CSMA/CA (Collision Avoidance): Used in Wi-Fi. Can't detect collisions on wireless. Instead: wait IFS period, then random backoff before transmitting. Optionally use RTS/CTS handshake.

Problems with Classful Addressing

The original IPv4 class system (A–E) caused catastrophic address waste and accelerated the exhaustion of the 4.3-billion address IPv4 space, leading to the invention of CIDR.

🥧

IPv4 Address Space Distribution

⚠️

Critical Problems

  • Massive Address Waste: Class boundaries are rigid. A company with 300 hosts must get a Class B (65,534 addresses), wasting 65,234 — 99.5% of the block.
  • IPv4 Exhaustion: 128 Class A blocks consumed 50% of all IPv4 space. With the internet boom, address pools emptied rapidly. IANA exhausted its pool in February 2011.
  • Routing Table Explosion: No way to aggregate routes across class boundaries meant millions of entries in backbone routers, exceeding memory limits of the era.
  • No Flexibility: Needed 500 hosts? Too small for Class B (65K), too big for Class C (254). No in-between option existed.
  • Supernetting Not Possible: Adjacent Class C blocks couldn't be combined into a single route entry, preventing route summarisation.
✅

Solution: CIDR (Classless Inter-Domain Routing)

Variable-Length Prefix:
Any prefix /0 to /32 is valid. Need 500 hosts? Use /23 (510 usable hosts) — no waste.
Route Aggregation:
Four /24 blocks (192.168.0.0–192.168.3.0) can be advertised as one /22, reducing routing table size.
Slower Exhaustion:
CIDR allowed ISPs to hand out exactly what was needed, buying time until IPv6 adoption.
VLSM:
Variable-Length Subnet Masking lets a single allocated block be divided into subnets of different sizes within an organisation.

✅ Conserves IPv4 addresses
Thousands of devices share one public IP — essential for home/SMB networks.
✅ Security through obscurity
Internal topology is hidden; unsolicited inbound connections are dropped by default.
❌ Breaks end-to-end model
P2P, VoIP (SIP), and IPsec require special NAT traversal techniques (STUN, ALG).
❌ Not a replacement for IPv6
NAT is a workaround. IPv6 with 128-bit addresses (3.4×10³⁸) eliminates address scarcity entirely.

Fragmentation Process

When does fragmentation occur?
When a datagram is larger than the MTU of the next-hop link. E.g., Ethernet MTU = 1500 bytes. A 4000-byte datagram must be split into fragments ≤ 1480 bytes of data (20-byte header).
  1. Original datagram (e.g., 4000 bytes total = 20B header + 3980B data) arrives at a router where outgoing MTU = 1500 bytes.
  2. Router splits data into chunks: Fragment 1 = 1480 bytes data, Fragment 2 = 1480 bytes data, Fragment 3 = 1020 bytes data.
  3. Each fragment gets the original Identification value, its own header (20B), and the Fragment Offset field set to its position ÷ 8.
  4. MF (More Fragments) bit = 1 for all but the last fragment. Last fragment has MF = 0.
  5. All fragments travel independently to the destination. They may arrive out of order.
  6. Destination host reassembles the original datagram using Identification + Fragment Offset. If any fragment is missing, the entire datagram is discarded.

How traceroute uses ICMP

  1. Send probe packet with TTL = 1. First router decrements to 0, discards packet, returns ICMP Time Exceeded. IP of that router is revealed.
  2. Send probe with TTL = 2. Second router sends back ICMP Time Exceeded. Second hop identified.
  3. Repeat, incrementing TTL by 1 each round until the destination sends ICMP Port Unreachable (or Echo Reply). Full path is mapped.
  4. Timestamp of each reply reveals RTT to each hop — useful for diagnosing where latency or packet loss occurs.

DLL Sub-Layers & Key Functions

LLC — Logical Link Control
Multiplexes protocols above (IPv4, IPv6, ARP). Provides optional flow control and error control between the two endpoint DLL entities.
MAC — Media Access Control
Contains the unique 48-bit hardware address burned into the NIC. Controls access to the shared physical medium using CSMA/CD or CSMA/CA.
Framing
Encapsulates the IP packet (from Network Layer) by adding a header (dst/src MAC, EtherType) and a trailer (FCS/CRC) to create a frame.
Flow Control
Prevents a fast sender from overwhelming a slow receiver. DLL uses mechanisms like stop-and-wait, sliding window (Go-Back-N, Selective Repeat).

OSI vs TCP/IP Model

OSI Model (7 layers)
Theoretical reference model. Separates Presentation and Session from Application. Distinguishes LLC from MAC. Never fully implemented as a single protocol suite.
  • Developed by ISO in 1984 as a vendor-neutral standard.
  • Provides a clean separation of concerns; ideal for teaching.
  • Layers 5 & 6 rarely have distinct protocol implementations.
TCP/IP Model (4 layers)
Practical internet model. Application = OSI 5+6+7. Internet = OSI 3. Network Access = OSI 1+2. Transport remains the same.
  • Developed by DARPA in the 1970s. Powers the modern internet.
  • Simpler, implementation-first philosophy.
  • Does not define how physical or data link should work.
